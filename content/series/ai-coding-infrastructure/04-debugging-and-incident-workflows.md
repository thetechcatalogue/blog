## Incidents Need Navigation, Not Just Search

In a live incident, the first problem is usually orientation. Engineers need to know where the entry point is, which services connect, and what recent changes are likely involved.

## Agent Assistance Fails Without Topology

If the agent only has local snippets, it often misses indirect dependencies, queue boundaries, or downstream side effects.

## Good Incident Tooling

Strong workflows connect alerts, anchor points, code ownership, blast radius, and troubleshooting guides into one path for fast diagnosis.

## Why This Matters

The value is not just better answers. It is shorter time to root cause and more consistent debugging under pressure.

## Summary

AI-assisted debugging works best when the system already knows its important pathways before the incident begins.
