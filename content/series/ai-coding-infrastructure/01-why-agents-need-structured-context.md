## Raw Code Is Too Expensive to Reconstruct Repeatedly

Coding agents can read files and search symbols, but repeating that process for every question is slow, noisy, and expensive.

## The Core Problem

A large repository contains too much structure to rediscover from scratch every time. Entry points, boundaries, integration paths, and critical business logic are not equally important.

## Structural Context Beats Flat Context

An agent works better when it receives curated, high-signal context instead of a giant dump of source code. That means knowing what matters before generation starts.

## What Good Context Looks Like

Useful context identifies key files, important symbols, security boundaries, runtime edges, and domain concepts that appear across many user tasks.

## Summary

AI coding quality improves when context is compressed around structure and importance, not just token count.
